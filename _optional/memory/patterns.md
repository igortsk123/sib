# Patterns — библиотека решений

> Переиспользуемые архитектурные шаблоны типовых фич/экранов. Перед новой фичей проверь — нет ли
> готового паттерна. Экономит решения и держит единообразие.

## Формат записи
```
## Pattern: <имя> (напр. list-with-filters)
**Применимость:** какие фичи/экраны подходят.
**Foundation-структура:** что нужно (типы, данные, слой доступа, маршрут, i18n).
**Scope:** что входит в паттерн, что оставляется на конкретную реализацию.
**Пример:** ссылка на реальную реализацию в коде.
```

---

## Pattern: list-with-filters (пример)
**Применимость:** списки сущностей с фильтрами, поиском, пагинацией.
**Foundation-структура:** тип элемента + тип фильтров; функция загрузки списка в слое доступа;
маршрут/страница-обёртка; ключи i18n.
**Scope:** входит — таблица, фильтры, пустое/загрузка/ошибка; не входит — конкретные колонки фичи.
