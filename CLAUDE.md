# sib — Агрегатор и распознавание гарантийных писем ДМС

Веб-продукт для медицинской клиники: автоматически собирает из почтовых ящиков письма страховых
компаний по ДМС («гарантийные письма»), распознаёт их (тело письма, PDF, Word, архивы+пароль, OCR),
сводит в единый проверяемый реестр и даёт сотрудникам поиск по пациентам/полисам/страховым + экспорт
в Excel для импорта во внутреннюю систему клиники.
Стек: Next.js + TypeScript (полностью типизировано, Zod-валидация на границах) + Tailwind/shadcn; PostgreSQL + Drizzle; Inngest
(durable-задачи и ретраи внешних вызовов); почта — Яндекс IMAP/SMTP XOAUTH2; LLM — OpenAI (через прокси).
Стадия: **greenfield, до-MVP** — каркас памяти развёрнут, кода ещё нет. MVP-петля (`product_brief.md`):
тестовый Яндекс-ящик → IMAP-забор (вкл. двойную пересылку) → распознавание полей → реестр+карточка →
Excel. Где сейчас — `.memory_bank/project-state.md`.

## Иерархия памяти

## Критично — инженерная конституция (`.claude/rules/engineering-principles.md`)
- **План first, code second** (`.claude/rules/agent-workflow.md`). Без явного «деплой» код не пишем.
- **Самопроверяемость:** владелец НЕ ревьюит код построчно. «Готово» = зелёный гейт
  (typecheck + lint + unit + integration + e2e + build) + e2e на затронутый критический поток.
- **Маленькие вертикальные срезы:** одна польз. способность UI→БД, с тестами и observability.
- **Гипотезы, не аксиомы:** любой контракт/схема/выбор — допущение; проверь, при расхождении
  отклонись и запиши ADR в `decisions.md` (контекст→решение→последствия).
- **Внешние вызовы — всегда в обёртке:** типизированный результат, ретрай через Inngest,
  заданное user-facing состояние ошибки.
- **Спрашивай владельца только по списку** `core/human-decisions.md`; остальное решай сам + фиксируй.
- **ПДн/мед.тайна:** данные пациентов — повышенные требования (`.claude/rules/guardrails.md`,
  `core/roles-and-access.md`).
- При конфликте источников — `source-of-truth.md` решает; расхождение фиксируем явно, не молчим.
- Память обновляется по `.claude/rules/memory-discipline.md` (меняется архитектура/контракты → обновить).

## Path-scoped правила (.claude/rules/)
- `agent-workflow.md` — workflow план→деплой (всегда).
- `memory-discipline.md` — когда сохранять/извлекать/синхронизировать память (всегда).
- `engineering-principles.md` — инженерная конституция: срезы, тесты, гейт, обёртки внешних вызовов (всегда).
- `guardrails.md` — protected-paths, ПДн/секреты (всегда).
- `code-standards.md` — TS/React стандарты (`**/*.{ts,tsx}`).
- `ui-rules.md` — Tailwind/shadcn, UI (`app/**`, `components/**`).
