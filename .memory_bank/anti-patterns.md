# Anti-patterns — грабли sib

> Чеклист при ревью среза. Дополняй по мере находок (особенно от кодогена/LLM).

## 1. Доверять техническому отправителю пересланного письма
**Симптом:** в GuaranteeLetter попал email того, кто переслал, а не страховой.
**Convention:** извлекать **исходного** отправителя/дату/тему из блока пересылки (ADR D5).
**Детект:** From письма = разработчик/клиника, а не домен страховой.

## 2. Авто-удаление дублей
**Симптом:** код удаляет письмо/запись по совпадению.
**Convention:** дубли только **помечать**, показывать в админке (ADR D6). Не удалять.
**Детект:** `delete` по сигналам дубля.

## 3. Голый внешний вызов
**Симптом:** прямой `imap` / `fetch` / `openai(...)` в server action/компоненте без обёртки.
**Convention:** обёртка — типизированный результат + Inngest-ретрай + user-facing ошибка (конституция §5).
**Детект:** сетевой вызов вне `lib/`/Inngest-функции; `try/catch`, молча глотающий ошибку.

## 4. ПДн/секрет в логе
**Симптом:** ФИО/полис/диагноз/токен в `console.log`/трейсе.
**Convention:** маскирование PII; секреты не логируем (`.claude/rules/guardrails.md`).
**Детект:** лог объекта письма/пациента целиком.

## 5. Доверять форме LLM/внешних данных
**Симптом:** ответ OpenAI / тело письма используются без валидации схемой.
**Convention:** валидировать на границе (zod), фиксировать confidence, низкое → ручная очередь.
**Детект:** `JSON.parse(llm)` без схемы; прямое присвоение полей из распознавания.

## 6. Спека как аксиома
**Симптом:** поля/форматы из брифа захардкожены без проверки на реальных письмах.
**Convention:** гипотеза → проверка → ADR при расхождении (`source-of-truth.md`).
