# Админка — Tier 1 сводка

> Простой UI без МИС-логики: быстрый доступ к письмам + контроль качества распознавания.
> Закрытый вход (медданные), мультитенант: платформенный админ → клиники → сотрудники, RBAC.

## Реальные страницы (`app/(admin)`, сверено 2026-07-22)
- **`/login`** — вход по телефону→коду Telegram (`doconpro_bot`).
- **`/`** — редирект по роли: платформенный админ → `/admin/clinics`; владелец → `/staff`; иначе `/registry`
  (отдельного дашборда со счётчиками нет).
- **`/admin/clinics`, `/admin/clinics/[id]`** — платформенный админ: клиники + назначение владельца.
- **`/staff`** — владелец: сотрудники, роли (dms/doctor/registry/registry_senior), инлайн-правка почты.
- **`/insurers`** — справочник страховых (название, домены, «не распознано парсером», статус).
  **`/insurers/[id]`** (платформенный админ) — шаблоны типов документов (`doc_template`, 27 образцов:
  образец→LLM-эталон→парсер) + журнал разбора по типу (дрейф).
- **`/registry`** — таблица ГП: поиск (ФИО/полис/№ГП), фильтры период/страховая/статус/направление/
  источник/«только проверка», «Выгрузить в Excel». Скоуп по клинике.
- **`/registry/[id]`** — карточка: распознанные поля + блок «Источники для сверки» (оригинал через
  `/api/original`), метод разбора, баннер «Проверить» при `needsReview`, «Сообщить об ошибке».
  (Кнопок «Подтвердить/Связать» и per-field confidence в UI нет — детали в product-spec §22.)
- **`/parse-log`** — журнал распознавания: чем разобрана запись, что парсер не нашёл (добрал ИИ).
- **`/error-reports`** (платформенный админ) — сообщения об ошибках; при «исправлено» → письмо автору.

## Excel-экспорт (`/api/registry/export`, exceljs)
Реестр ГП по текущим фильтрам, стабильный порядок колонок (ID записи, Пациент, ДР, Страховая, Полис,
№договора, №ГП, №обращения, Тип, Направление, Статус, Дата письма, Получено, даты, Ограничение, Услуги,
Источник, Метод, ссылка, **Дубль** — в конце, D20). Пустые ячейки типизированы (D18, `cellText`):
«нет данных» (absent) / «не распознано» (unreadable). В реестре дубль — значок ⧉, в карточке — ссылка
на первую запись. ⚠️ Утверждённый шаблон импорта клиники — **открытый вопрос** (`core/human-decisions.md`).
