# Админка — Tier 1 сводка

> Простой UI без МИС-логики: быстрый доступ к письмам + контроль качества распознавания.

## Экраны (§9)
- **Главный:** счётчики (новые, распознанные, на ручной проверке, архивы без пароля, пароли без архива,
  отказы, согласования за период) + последние обработанные.
- **Реестр** (главная таблица): дата письма/получения, страховая, ФИО, ДР, полис, статус согласования,
  услуги, срок, № письма, источник, статус проверки, ответственный, комментарий. **Фильтры:** период,
  страховая, статус, пациент, полис, услуга, врач, ящик, тип документа, статус распознавания, «только ручная проверка».
- **Карточка письма:** поля (с уверенностью по каждому), исходный текст, список вложений, просмотр
  PDF/Word, результат OCR, история распознавания, комментарии; кнопки «Подтвердить», «На ручную
  проверку», «Связать с другим письмом», «Скачать оригинал».
- **Очередь ручной проверки:** низкая уверенность, нет ФИО/полиса, архив без пароля / пароль без архива,
  не понята страховая, дубли, конфликты, несколько пациентов/писем в одном файле.
- **Справочники:** страховые (варианты названия, домены, правила); пользователи; почтовые ящики.

## Excel-экспорт (§10)
Единый реестр в Excel: фильтры (период/страховая/статус/только проверенные/все), выбор колонок,
**стабильная структура колонок**, ID записи, ссылка на оригинал, дата формирования. Колонки — §10.
⚠️ Утверждённый шаблон импорта клиники — **открытый вопрос** (`core/human-decisions.md`).
