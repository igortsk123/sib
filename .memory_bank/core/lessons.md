# Lessons — плейбук уроков (что НЕ сработало и почему)

> **Протокол обновления (ACE): ТОЛЬКО инкрементальные правки bullets** — добавить новый, уточнить
> существующий, пометить устаревшим (`~~текст~~ (устарело: причина)`). Файл целиком НЕ переписывать
> и НЕ «сжимать в сводку»: перезапись схлопывает детали, ради которых плейбук существует
> (в отличие от `project-state.md`, который по дизайну переписывается). CLEANUP_POLICY: для этого
> файла COMPRESS запрещён, только MERGE соседних дублирующихся bullets.
>
> **Когда писать:** проваленный/отброшенный подход, грабля с неочевидной причиной, «казалось X,
> оказалось Y» — сразу (через блокнот) или на `/memory-check`; поле «Уроки» из Completion summary
> каждого плана (особенно `partial`/`cancelled`) переносится сюда.
> **Когда читать:** ОБЯЗАТЕЛЬНО перед составлением каждого плана (`agent-workflow.md`, Этап 1) —
> иначе плейбук пишется, но не работает.

## Формат bullet
```
N. [YYYY-MM-DD] <ситуация> → пробовали <подход> → не сработало, потому что <причина> →
   правило: <что делать/не делать впредь>
```
