# Lessons — плейбук уроков (что НЕ сработало и почему)

> **ACE-протокол: ТОЛЬКО инкрементальные правки bullets** (добавить/уточнить/`~~устарело~~`).
> Целиком НЕ переписывать и НЕ сжимать (CLEANUP_POLICY: COMPRESS запрещён, только MERGE дублей).
> **Писать:** грабля/отброшенный подход — сразу или на `/memory-check`; «Уроки» из Completion summary
> планов. **Читать: ОБЯЗАТЕЛЬНО перед каждым планом** (`agent-workflow.md`, Этап 1).
> Формат: `N. [дата] <ситуация> → <подход> → не сработало: <причина> → правило: <как впредь>`.

## Уроки

1. [2026-07-23] Писал регэксп-правила по красивому `pdftotext -layout` → на бою промахи: парсер
   извлекает БЕЗ `-layout` (поточный порядок другой) → правило: правила/отладку писать ТОЛЬКО на
   том тексте, который даёт боевой `pdf_text()`; валидировать на нём же.
2. [2026-07-23] Казалось «LLM-значения = эталон» → офлайн-сверка показала: LLM клала период ГП в
   coverageFrom/To (по схеме там период ПОЛИСА; срок ГП живёт в validUntil) → правило: «золотой
   корпус» сверяет ФОРМУ, семантику полей решает схема+ADR, mismatch ≠ автоматически ошибка правила.
3. [2026-07-23] Формы-бланки (СОГАЗ-направления): в поточном тексте ЗНАЧЕНИЯ идут НАД
   подписями-лейблами («22.01.2026 22.02.2026 \n Дата начала…»), и пара дат стоит перед парой
   лейблов → правило: для форм пробовать паттерн «значение(я) ПЕРЕД лейблом», не только после.
4. [2026-07-23] Файл-first «в лоб» (любое значение из файла бьёт тело) → Балта: слабый FIO2-фолбэк
   из PDF перебил сильную метку «на имя …» из тела → правило: приоритет источника НЕ выше приоритета
   СИЛЫ извлечения (метка > маска > позиционный фолбэк); сначала сила, потом источник.
5. [2026-07-23] Шапка табличного файла как ДАННЫЕ: повтор заголовка перед каждой секцией
   (Ингосстрах «Полис|Фамилия|…», Энергогарант «Полис ДМС») парсился как пациент/полис →
   правило: значения, равные именам колонок, режутся в источнике (FIO_BAD/явные гварды), а колонка
   «Дата открепления» В ШАПКЕ не делает документ откреплением — тип решает ИМЯ ФАЙЛА/титул.
