# Распознавание — Tier 1 сводка

## Форматы (§5)
Тело (plain+HTML) · PDF: текстовый→прямое извлечение, скан→OCR/vision · DOC/DOCX→текст (оригинал
хранить) · Архив ZIP/RAR/7Z→распаковка при пароле · Изображения→OCR.
**Архив+пароль в разных письмах:** сопоставлять по отправителю/страховой/теме/времени/ФИО/№обращения/
№полиса + спец-правила. **Ручной fallback обязателен** (очередь связывания/ввода пароля).

## Поля
Пациент, страховая, согласование (статус/услуги/срок/лимит/условия), документ (тип/№/дата/источник/
ссылка). Перечень — `core/data-model.md` / `domain/product-spec.md` §7,13.

## Метод (§11)
Regex + правила по страховой + словари + **LLM** (OpenAI) + OCR/vision для сканов. Каждое поле — с
**уверенностью** (`confidence` jsonb) и ссылкой на фрагмент. Страховые → единый справочник.

## Контроль качества → ручная очередь
Проверять: ФИО/полис/страховая/статус/срок найдены; не истёк; не дубль; не конфликт; есть оригинал.
Низкая уверенность / нет ключевых полей → **очередь** (`needsReview`), не в реестр. ⚠️ Правила страховых —
**гипотезы** (§18.2): проверять на письмах, расхождение → ADR.

⚙️ **Подход (ADR D10):** детерминированное ядро (раскрутка пересылки/base64, страховая по реестру доменов,
извлечение по форматам) + **LLM** (OpenAI прокси, рантайм `gpt-5.4-mini`, Structured Outputs) только
на поля; **gpt-5.5** — учитель-эталон. Наблюдаемость гибрида — `parse_log` (поле «уехало в LLM» = источник
сменил форму → правим правило).

⚙️ **ФАЙЛ-FIRST (ADR D19, 2026-07-23):** свободный путь двухпроходный — поля сначала из ВЛОЖЕНИЙ, тело —
фолбэк; ФИО: сила извлечения (метка) важнее источника; тип enroll/detach — имя файла (прикр/prikr↔откр/snyat)
приоритетнее keywords; `source` честный. **field_status (D18):** absent=«нет данных» (искали файл+тело),
unreadable=«не распознано» (тех.сбой) — парсер эмитит, `enrich.prune_fs` снимает с дозаполненных.
Дубли (D20): пере-отправка списков → `is_duplicate` на приёме (dupKey).
