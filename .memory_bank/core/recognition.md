# Распознавание — Tier 1 сводка

## Извлечение текста по форматам (§5)
- Тело письма (plain + HTML).
- PDF: текстовый → прямое извлечение; скан/картинка → **OCR**.
- DOC/DOCX → извлечение текста (оригинал хранить).
- Архив (ZIP/RAR/7Z): распаковка при наличии пароля → обработка файлов внутри.
- Изображения → OCR.

## Архив + пароль в разных письмах
Сопоставлять архив и письмо-с-паролем по: отправитель, страховая, тема, близость по времени, ФИО/№
обращения/№ полиса, спец-правила страховой. **Ручной fallback обязателен** (очередь ручной обработки:
спец-ст вручную связывает письма / вводит пароль).

## Поля
Пациент, страховая, согласование (статус/услуги/срок/лимит/условия), документ (тип/№/дата/источник/
ссылка на оригинал). Полный перечень — `core/data-model.md` / `domain/product-spec.md` §7,13.

## Метод (комбинация — §11)
Regex + правила по страховой + словари/справочники + **LLM** (OpenAI) для сложных + OCR для сканов.
Каждое поле — со **значением уверенности** и ссылкой на фрагмент-источник. Названия страховых → единый справочник.

## Контроль качества → ручная очередь
Проверять: найдены ли ФИО/полис/страховая/статус/срок; не истёк ли; не дубль; не конфликт; есть ли
оригинал. Низкая уверенность / нет обязательных полей → запись в **очередь ручной проверки**, не в реестр.
⚠️ Форматы-правила страховых — **гипотезы** (§18.2): проверять на реальных письмах, расхождение → ADR.

⚙️ **Подход (ADR D10):** детерминированное ядро (раскрутка пересылки/base64, идентификация страховой по
реестру доменов, извлечение текста по форматам) + **LLM (OpenAI прокси, `gpt-5.4-mini`, Structured Outputs)**
только на извлечение полей. Эмпирика корпуса (13 страховых → домены, форматы, структура) — `domain/insurer-recognition.md`.
Карта срезов — `plans/recognition-roadmap.md`.
