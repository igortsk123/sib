# Человеческие решения — что спрашивать у владельца

> Принцип (ADR D3): спрашивай владельца **только** по пунктам ниже. Остальное (контракты, схемы,
> библиотеки, тесты) — решай сам, ADR в `decisions.md`. Не из списка → гипотеза+проверка; тупик → эскалируй.

## Открытые вопросы (разбор — `domain/product-spec.md` §19, §23)
- **Ящики:** сколько; провайдеры; IMAP/веб; app-password; помечать после обработки; архив или новые.
- **Тестовые письма:** сколько; примеры каждого типа/страховой; хранить ли цепочку; несколько пациентов в одном.
- **Данные / Excel:** обязательные поля; **утверждённый шаблон Excel** импорта; № карты из МИС; распознавать ли
  диагнозы/суммы/коды услуг; сопоставление со справочником.
- **Роли:** кто правит распознанное; видит ли врач оригиналы; кто выгружает Excel; «только просмотр»; 2FA.
- **Интеграции:** куда импортируют Excel и требования; API; уведомления о письмах.

## Безопасность / оператор данных
- ✅ **Хостинг ПДн (2026-06-21):** целевой = собственный РФ-сервер клиники; **полное логирование разрешено**
  (ADR D10), секреты в логи не пишем. Прод сейчас — интерим на общем VPS 193.160.208.41; перенос — при
  выходе из теста (ADR D4). Осталось: требования к ПДн; сроки хранения; шифрование на диске; бэкапы.

## Доступы / деньги / приоритеты
- Прод-хостинг и оплата; доступ к рабочим ящикам; платные лимиты OCR/LLM при росте; порядок срезов для демо.
- **SMTP наружу** (Яндекс XOAUTH2, уведомление автору): нужен рабочий OAuth-токен + проверка связи;
  пока письма pending → **согласовать с владельцем**.
