# Архитектура распознавания — детерминированный конвейер

> Принцип: **на каждом шаге — явное решение по признакам, не угадывание.** Письмо проходит фиксированный
> конвейер; на каждом узле — какой домен (страховая), какой тип письма, какой метод извлечения.
> Подтверждено на корпусе: **100% писем разбирается** (70 записей; ФИО 69/70; needsReview 1).

## Конвейер (по шагам, слева направо)
```
[0 Ingestion] IMAP-забор → сохранить оригинал .eml + вложения на сервере (инвариант: ссылка на источник)
   ↓
[1 ОПРЕДЕЛИТЬ СТРАХОВУЮ (домен)] по реестру InsuranceCompany.domains[]
   • основной признак: домен отправителя оригинала (после раскрутки пересылки)
   • фолбэк: домены в теле/Received — но ТОЛЬКО совпадающие с реестром (в теле есть домены клиник!)
   • промах → очередь «не понята страховая» (LLM-фолбэк по названию)
   ↓
[2 ОПРЕДЕЛИТЬ ТИП ПИСЬМА] по признакам (тема/тело/вложения) → см. таблицу типов ниже
   ↓
[3 ВЫБРАТЬ МЕТОД ИЗВЛЕЧЕНИЯ] по типу+носителю → см. таблицу методов
   ↓
[4 ИЗВЛЕЧЬ ПОЛЯ] детерминированно (таблицы/заголовки) ИЛИ LLM (свободный текст) + confidence
   ↓
[5 СТАТУС] approved / denied / detach(открепить) / enroll(прикрепить) — по ключам + LLM
   ↓
[6 ВАЛИДАЦИЯ] кросс-чек код↔LLM; нет ключевых полей/низкая уверенность → needsReview (пометка человеку)
   ↓
[7 ДЕДУП] Message-ID/hash/№ГП — помечать, не удалять. Объединение источников (письмо+пароль) в одну карточку.
```

## Шаг 1 — Реестр страховых (домен → страховая)
Сид `InsuranceCompany.domains[]` (домен ≠ название!):
`calltravel.eu`→Балта · `ingos.ru`→Ингосстрах · `zettains.ru`→Зетта · `renins.com/renhealth.com`→Ренессанс ·
`luchi.ru`→Лучшее · `astrovolga.ru`→Астро-Волга · `reso.ru`→РЕСО · `vsk.ru`→ВСК · `sogaz.ru`→СОГАЗ ·
`energogarant.ru`→Энергогарант · `sovcomins.ru`→Совкомбанк · `rgs.ru`→Росгосстрах · `alfastrah.ru`→АльфаСтрахование.

## Шаг 2 — Тип письма (признаки → тип)
| Тип | Признаки (тема/тело/вложение) | Статус |
|-----|-------------------------------|--------|
| **Гарантийное письмо (ГП)** | «гарантийное письмо», «согласовано/гарантируем оплату», ГП №…; ПДн пациента | approved |
| **Отказ** | «отказ», «не согласовано» | denied |
| **Открепление / аннулирование** | «открепл», «аннул», «недействительн», «снять с обслуж», «расторж» | **detach** |
| **Прикрепление (список застрахованных)** | «прикрепл», «список застрахованных», «принять в работу списки» | **enroll** |
| **Письмо-пароль** (к архиву) | «пароль для/на открыти», нет вложения | — (не запись; вливается в карточку ГП) |
| **Уведомление/служебное** | «направляем…», «просьба не отвечать», нет данных | — (docType service) |

## Шаг 3 — Метод извлечения (носитель → метод)
| Носитель | Метод | Кто извлекает поля |
|----------|-------|--------------------|
| Тело письма (text/HTML) | HTML→текст | **LLM** (свободная вёрстка) |
| **PDF (текстовый)** | `pdftotext` | LLM |
| **PDF (скан)** | `pdftoppm`→PNG → **LLM-vision** | LLM-vision |
| **Excel-СПИСОК** (≥2 пациента, шапка ФИО/полис) | таблица по заголовкам | **детерминированно построчно** (1 письмо→N) |
| Excel-одиночное письмо (.xls/.xlsx) | `xlrd`(бинарный .xls) / zip-XML(.xlsx) → текст | LLM |
| **RTF** | `striprtf` (чистый Python) | LLM |
| DOC (бинарный) | libreoffice/catdoc на сервере (follow-up) | LLM |
| **ZIP (ZipCrypto, шифр)** | пароль из письма-пароля → расшифровать → PDF/xls → метод по носителю | по носителю |
| PNG-логотипы/печати | пропуск (не ГП) | — |

**МЕХАНИКА-ПЕРВОЙ (ADR-принцип):** авто-генерённые шаблоны страховщиков (PDF/Excel/RTF/DOC текстовые) —
разбираем **механически** (размеченные поля «Пациент:»/«Полис:»/«ГП №» по регэкспу + is_fio). Если механика
дала ФИО+полис/ГП → запись готова **без LLM** (дёшево, стабильно). **LLM — только фолбэк**, если механика не
справилась. Скан/картинка → vision. Per-insurer регэксп-шаблоны растут со временем (расширяют покрытие механики).
**Excel-СПИСОК:** колонка «Пациент» = авто-выбор колонки с максимумом валидных ФИО; в неё попадают **только
настоящие ФИО** (строгий is_fio: 2–3 заглавных кириллических слова, без цифр/двоеточий/меток/подписей).
**.doc** (бинарный Word) — `olefile` (поток WordDocument, UTF-16LE). **.xls** — `xlrd`. **RTF** — `striprtf`.
**Правило:** чистая таблица → по заголовкам (точно, без LLM); свободный текст → механика, не вышло → LLM; скан → vision.
**ФИО в ТЕМЕ письма:** у части страховых (Совкомбанк аннулирование) ФИО пациента — в Subject; тема включается
в текст распознавания. **Раздельные колонки** Фамилия/Имя/Отчество (СОГАЗ-списки) — склеиваем в ФИО.
**Шапка таблицы** = строка с МАКСИМУМОМ колоночных ключей (≥2), не первая попавшаяся с одним словом (проза
«…список застрахованных…» — не шапка). Список разворачивается даже с 1 пациентом при сильной шапке (≥3 ключа).
**МАСКИ ПОЛИСОВ (per-insurer):** полис часто НЕ подписан «полис», но имеет узнаваемый формат — ищем по маске
страховой (фолбэк, если метки нет): Зетта `ММ__-#######/##`, СОГАЗ `##LM####/######`, Ингосстрах `Пр########`,
Совкомбанк `###-##-######-##/#####`, Балта `########-SOS-#`, РЕСО `#######-####/##`, ВСК `#####G#######-##`,
Росгосстрах `##-М__-####-##/####-####`, Энергогарант `######-###-######-###`. Маски растут по мере примеров.
**Точность:** LLM возвращает confidence; кросс-чек с детерминированным; «точность важнее полноты» (сомнение→null).
**Пациент ≠ директор/адресат/обращение** («Уважаемая…», «Руководителю Учреждения», директор клиники): is_fio
отсекает их и в таблицах, и в FIO2-фолбэке. ФИО нормализуется «Фамилия Имя Отчество» + схлопывание переносов.

## Per-insurer (валидировано на корпусе)
| Страховая | Носитель ГП | Метод | Особое |
|-----------|-------------|-------|--------|
| Балта (`calltravel.eu`) | PDF | pdftotext→LLM | |
| Ингосстрах (`ingos.ru`) | PDF + .xls + **списки прикр/откр** | pdftotext/xlrd; списки построчно | png=логотипы |
| Зетта (`zettains.ru`) | **zip(шифр)→PDF** | unzip(пароль из письма-пароля)→pdftotext→LLM | архив+пароль = 1 карточка |
| Ренессанс (`renins.com`) | PDF + .xls + списки | pdftotext/xlrd | |
| Лучшее (`luchi.ru`) | **PDF во вложении** | pdftotext→LLM | тело — лишь уведомление |
| Астро-Волга (`astrovolga.ru`) | PDF | pdftotext→LLM | |
| РЕСО (`reso.ru`) | .xlsx (письмо-в-Excel) + RTF + списки | xlsx-текст/striprtf→LLM; списки построчно | |
| ВСК (`vsk.ru`) | PDF | pdftotext→LLM | |
| СОГАЗ (`sogaz.ru`) | **бинарный .xls** + PDF | xlrd→LLM | |
| Энергогарант (`energogarant.ru`) | PDF (+ doc + png) | pdftotext→LLM | doc — follow-up |
| Совкомбанк (`sovcomins.ru`) | PDF; аннулирование в теле | pdftotext/LLM | «аннулировать ГП №…»→detach |
| Росгосстрах (`rgs.ru`) | **zip(шифр)→.xls** + PDF | unzip(пароль `rgs…`/письмо)→xlrd→LLM | |
| АльфаСтрахование (`alfastrah.ru`) | **RTF** (ГП) + .xlsx (списки откр/прикр) | striprtf→LLM; списки построчно | cover-notes + аннулирование |

## Куда тяжёлое (прод)
Все шаги ≥2 — в **Inngest-функциях** (ретраи, идемпотентность), не в request-пути. Внешние вызовы (IMAP,
OpenAI-прокси, распаковка, OCR) — в обёртке (Result). Тулзы извлечения на сервере: pdftotext/poppler,
striprtf, xlrd, libreoffice(для doc/бинарного xls в проде) — добавить в Docker-образ при S1.
