# Domain — доменные модели (Tier 2)

> Полные доменные модели, не зависящие от UI/инфраструктуры. **Одна сущность = один файл**
> (`task.md`, `user.md`, …). Сводная карта моделей — в `core/data-models.md` (Tier 1).

## Соглашения
- Каждый файл — frontmatter (`tier: 2`, `topic`, `scope`, `tier1: ../core/data-models.md`, `updated`).
- Описывай: поля, типы/enum, инварианты, переходы состояний, связи с другими сущностями (`[[name]]`).
- Контракты API/форматы ответов — здесь или в отдельном `api-contracts` (Tier 2), на выбор проекта.
