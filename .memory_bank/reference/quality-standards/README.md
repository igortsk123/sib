# quality-standards — эталоны качества (project-agnostic)

Переносимые senior-эталоны: целевая планка качества для код-ревью, Definition of Done и CI-гейтов.
Формат каждого критерия — **цель + измеримый порог/проверка** + уровень (**MUST / SHOULD / PREFER**).
Это **целевое состояние**, а не описание текущего кода. Конкретику проект кладёт в overlay (см. ниже).

## Файлы
| Файл | Про что | Берём, если |
|------|---------|-------------|
| `backend_service.md` | веб-сервис/бэкенд: 21 сквозной аспект (SEC/PERF/PROTO/ISO/SCALE/TEST/OPS/FT/RES/PAT/MAINT/CONC/DATA/DR/REL/I18N/COST/DX/DOC/A11Y/PORT) + слои | есть серверная логика/API |
| `sql.md` | PostgreSQL/SQL: OLTP, OLAP/Kimball, DDL/целостность, перф, NULL-корректность, изоляция, время/TZ, JSONB, доступ/PII, миграции | есть БД |
| `web_frontend.md` | SPA/SSR-фронт: SEC/PERF/модульность/тестируемость/a11y/RESIL/CONC/I18N/Forms/Routing/SEO + слои | есть веб-UI |
| `android_app.md` | Android-приложение | есть Android-клиент |

## Как применять (важно — не «прочитать и забыть»)
1. **Overlay на проект.** Рядом завести `quality-criteria.<project>.md` (или `<area>-quality-criteria.<project>.md`):
   для каждой области — **зрелость 0–4** (0 нет · 1 точечно · 2 без enforcement · 3 ловит машина · 4 эталон),
   главный долг, привязка ID критерия (`SEC-4`/`O3`/`RESIL-1`) к месту в коде, таблица разрывов `current → target`.
   Сам эталон не правим — он чистый и переносимый.
2. **Применимость явно.** Каждая ★-секция / область — «применимо / N/A + обоснование». N/A примеры:
   OLAP — нет аналитики; i18n — одноязычный internal-инструмент; SEO — приватная зона за авторизацией.
3. **Сдвиг проверок влево.** Цель миграции каждой области — зрелость **3**: нарушение ловит машина
   (lint/CI/тест/EXPLAIN-gate/health), а не ревьюер. «Частично» не засчитывается — только зелёная проверка.
4. **Релиз-блокеры.** Перед 100% трафика — зелёные MUST релевантных областей (см. Definition of Done в каждом файле).

Как разбить внедрение на этапы и с чего начать — `_optional/memory/quality-criteria.md`.
