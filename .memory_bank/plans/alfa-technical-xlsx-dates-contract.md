## Цель
Точно извлекать у Альфы (и кросс-страховых): дату рождения (БАГ — писалась дата письма), номер
договора, период обслуживания/действия (с/по), ограничение (лимит/условия). Использовать технический
`*_all.xlsx` Альфы (машинные колонки). Добавить недостающие поля модели, вывести в реестр/экспорт.

## Источник задачи
Альфа: снятие/постановка на мед.обслуживание в Excel + ГП в RTF; КАЖДЫЙ Excel сопровождает техническим
`*_all.xlsx`. Баг: дату письма записали как дату рождения; сохранять реальную дату рождения и период
обслуживания (с/по). №F514Z/045/7393S/24 — номер договора. Астро-Волга: «направление в пределах суммы»
→ нужна колонка ОГРАНИЧЕНИЕ (где сумма, где манипуляции). Балта: ограничение в PDF «доп. информация»
(«Покрываем: … ВАЖНО! если хрон. диагноз/серная пробка — пациент платит сам»). ВСК: переносить период
полиса (с/по); метод стоит LLM — оценить детерминизм.

## НАЙДЕНО: структура технического файла Альфы (корпус `13_Альфа/02_2.xlsx`, заголовки)
Машинные колонки (en): `row_number, policy_number, date_from, date_to, date_cancel, prog_code, fio,
prog_name, add_phone, address, phone_home, phone_office, birth_date, person_sex, insurer, ag_number,
our_subj, hosp_extrim, hospital, ambulance`. Парный человеческий файл `02_1.xlsx` несёт «Договор № …».
Маппинг: `fio`→patientFullName, `birth_date`→patientBirthDate, `policy_number`→policyNumber,
`date_from`→coverageFrom, `date_to`→coverageTo, `date_cancel`→снятие (approvalStatus=detach + дата),
`prog_name`→services/программа. contractNumber — из «Договор №» парного файла (или ag_number — сверить).

## Ответы владельца
- Договор — отдельным полем `contractNumber`. ✅
- Период «с/по»; иногда только «до» (открытого нет) → обе даты nullable. ✅

## Скоуп — что входит
1. **Модель** (`guarantee.ts` + миграция): `contractNumber text`, `coverageFrom date`, `coverageTo
   date`. (`patientBirthDate`/`letterDate`/`validUntil`/`amountLimit`/`conditions` уже есть.)
2. **Технический `*_all.xlsx`** (`extract_dataset.py`): детект по сигнатуре заголовков (есть
   `policy_number`+`fio`+`birth_date`+`date_from`) → ПРЕДПОЧИТАТЬ как источник чистых строк для Альфы.
   Прямой маппинг колонок (см. выше). Никакого LLM для этих строк.
3. **Фикс дат (баг)**: даты — ТОЛЬКО из помеченных колонок/меток (`birth_date`, `date_from/to`,
   «дата письма»); убрать «первую дату из текста». letterDate больше не попадает в birthDate.
4. **Договор** (regex кросс-страховой + ячейка «Договор №»): `\bF[0-9A-ZА-Я]+/\d{3}/[0-9A-ZА-Я]+/\d{2}[0-9A-ZА-Я]*\b`
   (F514Z/045/7393S/24, F169W/045/0330S/23ППTOMSK). Excel(шапка)/RTF/тело.
5. **Ограничение** (НЕ новое поле — переиспользуем `amountLimit`+`conditions`): извлечение кладёт
   сумму → `amountLimit`, ограничивающий текст (Балта «Покрываем… ВАЖНО…», Альфа prog_name, манипуляции)
   → `conditions`. Экспорт/реестр: колонка **«Ограничение»** = amountLimit ⊕ conditions.
6. **Период полиса/обслуживания (с/по)** — общий `coverageFrom`/`coverageTo`: Альфа date_from/to,
   ВСК «полис с-по». Иногда только coverageTo.
7. **RTF**: дата рождения, период (с/по) и/или validUntil, номер договора (regex п.4).
8. **LLM-схема/промпт** (`enrich.py`): добавить `contractNumber`, `coverageFrom`, `coverageTo`; чётко
   развести letterDate↔patientBirthDate; просить ограничивающие условия в `conditions`.
9. **Сид-маппинг** (`corpus.ts`) + **реестр/экспорт**: дата рождения, договор, период, «Ограничение».
10. **Кросс-страховые**: дата-дисциплина + новые поля для всех; регресс на образцах корпуса.

## ВСК — оценка LLM → детерминизм (по запросу)
ВСК шлёт **текстовый PDF 2 стр.** (`pdftotext`, помечен «чисто»), полис — маска `\d{5}G\d{6,8}-\d{2}`.
Это РЕГУЛЯРНЫЙ структурированный текст → детерминизм РЕАЛЬНО возможен: полис по маске, даты по меткам
(«Срок действия полиса с … по …»), договор по regex. Сейчас LLM, потому что детерм-путь не label-aware
по датам. Вывод: добавить для ВСК детерм-извлечение дат-с/по + полис/договор (с дата-дисциплиной п.3),
LLM оставить фолбэком. Снижает зависимость от LLM, ускоряет, дешевле.

## Скоуп — что НЕ входит
- Боевой Inngest-рантайм IMAP (отдельный воркстрим). Семантика услуг.

## Шаги
1. Схема + миграция (contractNumber, coverageFrom, coverageTo). Гейт typecheck/test.
2. `extract_dataset.py`: технический-файл детект+маппинг; дата-дисциплина; колонки период/договор; regex договора.
3. RTF + ВСК-детерм (даты по меткам, полис/договор по маске/regex).
4. `enrich.py`: новые поля + разведение дат + ограничивающие условия в conditions.
5. `corpus.ts` маппинг; реестр/экспорт: дата рожд, договор, период, «Ограничение».
6. Прогон оффлайн-пайплайна → новый dataset.json → re-seed (прод — отдельная авторизация, с бэкапом).
7. Кросс-проверка на образцах (Альфа Excel+тех, РЕСО, Ингосстрах, ВСК, Балта, Астро-Волга).

## Риски
- **ПДн:** строки пациентов не выводить; работать по заголовкам/коду/структуре.
- **Дата-дисциплина** может обнулить даты без меток (лучше null, чем подмена) — там остаётся LLM.
- **Re-seed прода** перетирает реестр — по явной команде, с бэкапом.
- Кросс-страховые парсеры не ломать — изменения аддитивные + регресс.
- contractNumber источник (шапка «Договор №» vs ag_number) — сверить на образце при реализации.
