## Цель
Показать клинике рабочий продукт: реестр гарантийных писем из реального корпуса (51 письмо) с поиском,
карточкой со ссылкой на оригинал (на сервере) и выгрузкой в Excel. По ходу — править схему.

## Источник задачи
«Реестр ГП можешь создать, полностью распарсив текущие письма по схеме как решил. Покажешь клинике как
будто продукт готов, поиск работает, можно перейти по ссылке на каждое письмо и выгрузить в Excel.»

## Решения
- **Извлечение полей — детерминированно сейчас** (раскрутка из `domain/insurer-recognition.md`: insurer по
  домену, даты, № ГП из имени/текста, формат-источник; ФИО/полис/статус — регэкспы best-effort). **LLM-обогащение
  (ADR D10) — дроп-ин позже**, когда задан `OPENAI_API_KEY` (сейчас не задан). Excel-реестры → 1 письмо N строк.
- **Сидинг (one-time, корпус фиксирован):** Python извлекает датасет + раскладывает оригиналы (.eml + вложения)
  → загрузка на сервер в хранилище → TS-сид (Drizzle) вставляет email_message/attachment/guarantee_letter.
- **Хранилище оригиналов на сервере:** `/opt/sib-storage` ← том в контейнер `/app/storage` (как uploads sup2).
  Раздача — защищённый роут `/api/original/...` (только по сессии — ПДн).
- **Реестр UI:** таблица с поиском (ФИО/полис/страховая), фильтры; карточка письма с полями + ссылкой на
  оригинал/вложения; выгрузка Excel (exceljs).

## Скоуп — что входит
- Деплой: монтировать том `/opt/sib-storage:/app/storage` в `run_fe()` (deploy/sib-deploy.sh).
- `lib/storage.ts` (пути), `app/api/original/[kind]/[id]/route.ts` (guarded раздача файла).
- Python: `.mail-intake/extract_dataset.py` — раскрутка → JSON-датасет + папка originals (id-именование).
- TS-сид `lib/db/seed/corpus.ts` + `pnpm db:seed:corpus` — читает датасет, вставляет строки (идемпотентно).
- Реестр: `lib/server/registry/queries.ts` (поиск/фильтры), `/registry` таблица + поиск, `/registry/[id]` карточка.
- Excel-экспорт: `app/api/registry/export/route.ts` (exceljs, колонки бриф §10).

## Скоуп — что НЕ входит
LLM-извлечение (следующий шаг при ключе); IMAP; редактирование полей в карточке (read + ссылка пока).

## Критерии приёмки
- [ ] Реестр на проде наполнен из корпуса (≈51 писем → N записей), виден в /registry.
- [ ] Поиск по ФИО/полису/страховой работает.
- [ ] Карточка открывает оригинал письма и вложения по ссылке (guarded).
- [ ] Выгрузка в Excel работает.
- [ ] Гейт зелёный; схема поправлена, если поля не лезли (ADR при отклонении).
