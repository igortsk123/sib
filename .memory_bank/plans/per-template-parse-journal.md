## Цель (по указанию владельца)
Журнал разбора имеет смысл вести НЕ в контексте страховой, а **внутри каждого ШАБЛОНА** (тип документа),
т.к. распознавание ведётся строго в контексте шаблона. Иерархия: **страховая → домен → шаблоны → по
каждому шаблону журнал разбора** (где помог парсер, где только ИИ). Плюс — у каждой страховой завести
её уникальные шаблоны автоматически.

## Состав
1. **Предзаполнение шаблонов**: в сиде корпуса создать `doc_template` для каждой пары (страховая × тип
   документа), что реально встречается в данных. Идемпотентно (unique insurer+docType). Статус
   `parser_ready` (авто из корпуса; владелец может загрузить эталон → `llm_parsed`).
2. **Журнал внутри шаблона**: на `/insurers/[id]` в карточке каждого шаблона показать статистику по
   `parse_log`, отфильтрованную по (страховая × этот тип):
   - записей типа N; «Чем разобрано»: Парсер N · Парсер+ИИ N · ИИ N;
   - «Что добирал ИИ»: поля (detGap) по-русски (FIELD_HINTS);
   - дрейф (уже есть «N на разбор») — оставить.
3. **queries**: `templateJournalByType(insurerName)` → docType → {n, methods, gaps} из parse_log.
4. **/parse-log**: упростить до ГЛОБАЛЬНОЙ сводки (итоги по методам + общие пробелы), убрать таблицу
   «По страховым» (детализация теперь в шаблонах). Ссылка-подсказка «детально — в шаблонах страховой».
