## Цель
Закрыть вход (медданные — только заранее заведённые пользователи), дать платформенному админу назначать
админов клиник и управлять страховыми/доменами, а админу клиники — заводить сотрудников.

## Источник задачи
«Бот не должен пускать всех подряд — это медицинские документы. Пока сотрудник не добавлен — войти нельзя.
Я (платформенный админ) завожу клиники, админов клиник и сотрудников, добавляю страховые и домены.
Админ клиники — только сотрудников. Создал клинику — не могу назначить администратора/редактировать.»

## Решения
- **Закрытый вход (security).** Код и сессия — только для «известных» телефонов:
  `phone === BOOTSTRAP_ADMIN_PHONE` ИЛИ `app_user` с этим телефоном уже существует (его завёл админ).
  - `requestCode`: неизвестный номер → НЕ создаём попытку, НЕ шлём код, ответ «номер не зарегистрирован,
    обратитесь к администратору». (Бот выдаёт код только по существующей попытке → неизвестным не выдаст.)
  - `verifyCode`: неизвестный номер → отказ. **Убрать авто-создание** пользователя при входе; создаётся
    только bootstrap-админ (как платформенный) при первом входе.
- **Роль «Администратор клиники»** = `membership.role='owner'` (переименовать ярлык owner → «Администратор клиники»).
  Может только добавлять/блокировать сотрудников своей клиники.
- **Платформенный админ:**
  - страница клиники `/admin/clinics/[id]`: назначить **админа клиники** (по телефону) + добавлять
    **сотрудников**; список участников с ролями; блокировка.
  - **управление страховыми:** на `/insurers` — добавить страховую, добавить/убрать домен (редактируемый реестр).
- **Provisioning = создание входа.** Добавление админа/сотрудника создаёт `app_user` (если нет) → этот
  телефон становится «известным» → может войти. Пока не добавлен — войти нельзя (закрытый вход).

## Скоуп — что входит
- `lib/server/auth/actions.ts`: гейт `isProvisioned(phone)` в `requestCode`/`verifyCode`; `upsertUser` →
  не создаёт неизвестных (только bootstrap). Сообщения «не зарегистрирован».
- `lib/roles.ts`: ярлык owner → «Администратор клиники»; layout roleLabel.
- Клиники: `/admin/clinics/[id]` (детали) + экшен `addClinicOwner` (есть) + UI; список членов (новый query).
- Сотрудники платформенным админом: переиспользовать `addStaff` (гард `requireClinicOwner` уже пускает
  платформенного админа) на странице клиники.
- Страховые: экшены `createInsurer`, `addInsurerDomain`, `removeInsurerDomain` (только платформенный админ) + UI на `/insurers`.
- Тест на `isProvisioned`/гейт входа.
