## Цель (правило владельца)
Боевой приём писем (S1) должен быть «самовосстанавливающимся»: ничего не теряем, а пробелы/сбои сами
сигнализируют админу и заводят задачу на парсер.

## Сценарии
1. **Известный домен, НЕТ шаблона типа** (напр. письмо с `alfastrah.ru`, но такого типа документа ещё нет):
   - всё равно создать запись в реестре (НЕ отбрасывать);
   - подсветить строку жёлтым (`needsReview`);
   - разобрать через **LLM** (fallback);
   - **авто-создать `doc_template`** для (страховая × тип) с этим письмом как образец (тема+тело+файл), статус `new`;
   - **алерт админу**: системный `error_report` («новый тип у <страховая>, нужен парсер») + письмо на почту (SMTP).
2. **Сбой/дрейф** (поле, что раньше брал парсер, перестало париться → добрал LLM, или статус неизвестен):
   - запись остаётся, строка жёлтая, поле берёт LLM;
   - `parse_log` фиксирует `detGap` → видно в журнале шаблона;
   - алерт админу (error_report + почта) «парсинг сломался у <страховая×тип>».

## Что уже готово (опоры)
- `parse_log` + журнал по шаблону (метод, detGap) — таблица и UI готовы ПРИНИМАТЬ авто-записи.
- `error_report` + страница `/error-reports` + `notifyErrorFixed` — канал алертов (системные репорты).
- `doc_template` (insurer×docType, образец тема/тело/файл) — авто-создание строкой `onConflictDoUpdate`.
- Маппинг домен→страховая (insurer.domains), классификация типа (status_of/docType), careType.

## Что нужно (зависит от S1 + SMTP)
- **Живой пайплайн приёма** (IMAP S1, Inngest): на каждое письмо — det-парсер → LLM-fallback → ЗАПИСЬ
  `parse_log` (insurer = ОФИЦИАЛЬНОЕ имя, docType, method, detGap) → запись реестра (+needsReview при fallback).
- **Авто-создание шаблона**: если (insurer×docType) нет в `doc_template` — вставить (образец = это письмо).
- **Системный алерт**: создать `error_report` (тип «system»/«new_template»/«parse_drift») + отправить письмо
  админу (нужен **SMTP**, сейчас только лог).

## Файлы (при S1)
`lib/server/ingest/*` (новый — IMAP+Inngest), `lib/server/recognition/*` (порт логики из `.mail-intake`),
`lib/server/error-reports/actions.ts` (+системные алерты), `lib/server/mail/notify.ts` (+SMTP),
`doc_template` авто-insert, `parse_log` live-insert.

## Зависимости
S1 (IMAP-забор + тулзы извлечения в Docker) + SMTP. Без них журнал/алерты пишутся только из корпуса (сид).
