# _intake/brief — цель и описание проекта

> Это одна из двух **входных** папок. Положи сюда всё, что описывает проект **как он задуман**.
> Команда `/memory-init` прочитает это и сгенерирует `product_brief.md`, `project-state.md`,
> заполнит плейсхолдеры в `CLAUDE.md` и предложит стартовые темы для `core/`.

## Что сюда класть (любой формат: .md, .txt, скриншоты)

- **Цель продукта** — какую проблему решаем, для кого, зачем.
- **Описание** — что это, тип проекта (web/mobile/cli/lib/backend), стадия (MVP/прод).
- **Целевая аудитория** — кто пользователь, контекст использования.
- **Scope** — что входит в текущий этап и что **явно не входит**.
- **Стек / ограничения** — язык, фреймворки, инфраструктура, нефункциональные требования.
- **Метрики успеха** — как поймём, что цель достигнута.
- **Ссылки** — прод-URL, репозиторий, дизайн, тикеты (если есть).

## Что НЕ сюда

- Историю изменений, старый код, транскрипты → клади в `../history/`.
- Чувствительные секреты (токены, пароли) → им не место в memory bank вообще.

## После обработки

`/memory-init` перенесёт исходники в `_intake/_processed/` и проставит в сгенерированных
доках `source:` со ссылкой сюда (provenance). Можешь удалить `_processed/` позже вручную.

> Минимум для запуска: один файл `goal.md` с парой абзацев. Чем больше контекста — тем
> точнее соберётся memory bank, но переусердствовать не нужно: дубли потом чистятся аудитом.
