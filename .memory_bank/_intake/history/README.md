# _intake/history — история проекта

> Вторая **входная** папка. Положи сюда всё, что описывает проект **как он развивался**.
> `/memory-init` извлечёт отсюда принятые решения (→ `decisions.md` в ADR-формате), уже
> сделанное (→ `project-state.md`), доменные сущности и legacy-сценарии (→ `domain/`).

## Что сюда класть (любой формат: .md, .txt, .json, экспорт чатов)

- **Changelog / история релизов** — что и когда менялось.
- **Старые доки и спеки** — предыдущие версии архитектуры, ТЗ.
- **Транскрипты** — выжимки обсуждений, решений, договорённостей.
- **Legacy-заметки** — что в старом коде важно для понимания домена.
- **Накопленные грабли** — повторяющиеся ошибки (станут `anti-patterns.md`, если активируем модуль).
- **Готовые куски архитектуры** — описания API, моделей, флоу.

## Как это используется

- Принятые решения с обоснованием → записи ADR в `decisions.md`.
- «Что уже готово» → секция в `project-state.md`.
- Доменные модели → файлы в `domain/` (Tier 2).
- Legacy, который **нельзя тащить как шаблон**, → фиксируется в `source-of-truth.md`
  (раздел «Невалидные источники»).

## Что НЕ сюда

- Цель и описание задуманного → `../brief/`.
- Секреты → не в memory bank.

## После обработки

Исходники уезжают в `_intake/_processed/`; `source:` в сгенерированных доках ссылается сюда.

> Если истории нет (новый проект) — оставь папку пустой, `/memory-init` это учтёт и соберёт
> memory bank только из `brief/`.
