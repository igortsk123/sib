# Source of Truth — доктрина истины

Источники делятся на два непересекающихся домена. При конфликте — решаем по приоритету внутри
домена; если домены спорят между собой — фиксируем расхождение явно, не «забалтываем».

## Домен кода — что реализовано
Приоритет по убыванию:
1. Текущий codebase (то, что реально работает). *(greenfield — пока пусто)*
2. `project-state.md` — снимок состояния.
3. `core/data-model.md` + `domain/product-spec.md` — спека-**гипотеза** (см. ниже).
4. `domain/*` — доменные модели.
5. Legacy — нет (greenfield).

## Домен процесса — как работать
Приоритет по убыванию:
1. `.claude/rules/agent-workflow.md` + `.claude/rules/engineering-principles.md` — протокол план→деплой
   и инженерная конституция (самопроверяемость, срезы, гейт, обёртки).
2. `.memory_bank/plans/` — активные планы.
3. `guides/code-standards.md`, `guides/review-rules.md`, `quality-criteria.md`.
4. `decisions.md` — принятые решения (ADR).

**Важно:** код не отменяет процесс. «Код уже написан иначе» — не основание игнорировать workflow.

## Спека = гипотеза, не аксиома (ключевой принцип)
`domain/product-spec.md` — это идея продукта v0.1. **Любой контракт/поле/формат/правило страховой из
спеки — это допущение**, которое проверяется на реальных тестовых письмах. При расхождении:
не подгонять реальность под спеку, а **отклониться и записать ADR** (`decisions.md`: контекст→решение→
последствия) + обновить `core/*`. Бриф §19 — открытые вопросы, ответы у владельца (`core/human-decisions.md`).

## Как трактовать конфликты
- Реализация расходится со спекой → проверить на реальных письмах; верное фиксируем ADR.
- Решение из `decisions.md` устарело → не молча игнорировать, а добавить новую ADR, отменяющую старую.
- Модель пропустила planning step / гейт → это нарушение процесса, а не оптимизация.

## Невалидные источники (что НЕЛЬЗЯ тащить как шаблон)
- Бриф v0.1 как жёсткие аксиомы (это гипотезы; форматы страховых меняются — §18.2).
- Поля/форматы страховых «на глаз» без проверки на реальных письмах.
- Архитектура проекта **sup2** как шаблон (другой домен — портала поставщиков). Переиспользуем
  только **паттерн Яндекс-почты XOAUTH2** (`_secrets/ACCESS.md`), не структуру.
