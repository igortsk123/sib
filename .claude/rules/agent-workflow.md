# Workflow агента — режим AUTOPILOT

**AUTOPILOT (включён владельцем):** агент работает **автономно** — планирует и **сразу выполняет и деплоит**,
НЕ ожидая команды «деплой» на каждый шаг. Обратимые решения принимает сам и фиксирует. Полная версия —
`guides/agent-workflow.md`. (Copilot-режим «план → жди деплой» отключён.)

## Что остаётся ОБЯЗАТЕЛЬНЫМ и в autopilot
1. **Гейт перед «готово»:** typecheck+lint+unit+integration+e2e+build — зелёный. Красный = не готово (не деплоим).
2. **Память:** архитектурные изменения/решения → Memory Bank (`memory-discipline.md`) + `/memory-check` (audit «чисто»).
3. **Эскалация только по `core/human-decisions.md`** — эти вопросы к владельцу. Остальное решаем сами (обратимая гипотеза → проверить → зафиксировать ADR).
4. **Guardrails ABSOLUTE:** секреты/ПДн/read-only почты — не нарушать (`.claude/rules/guardrails.md`).
5. **Крупные необратимые/внешние действия** (снос прод-данных, массовая рассылка наружу, смена инфры) — коротко предупредить/подтвердить, если владелец явно не санкционировал.

## Процесс (autopilot)
1. Прочитать `CLAUDE.md` + файлы по теме (INDEX импортирован — уже в контексте; decision tree).
2. **Свериться с `core/lessons.md` и `anti-patterns.md`** — не повторять отброшенные подходы.
3. Для нетривиальной задачи — план-файл `plans/<slug>.md` (`draft`→`in_progress`); для мелких правок — можно без файла.
4. **Выполнить сразу** (не ждать «деплой»): срез UI→БД, тесты, гейт зелёный, деплой.
4b. Крупная задача (>5 файлов или новая подсистема) → read-only субагент `verify`
   (`.claude/agents/verify.md`): план vs diff + выборочная сверка `core/*` с кодом; расхождения
   исправить ДО `/memory-check`.
5. Память при изменении архитектуры + `/memory-check` (audit «чисто»); отброшенные подходы →
   `core/lessons.md` (инкрементальные bullets) и «Уроки» в summary плана.
6. Готово → `completed` + дата + summary → `completed_plans/`. Частично → `partial`.

**Гейт:** план НЕ становится `completed`, пока гейт не зелёный, `/memory-check` не выполнен и audit не «чисто».

## Статусы плана
| Статус | Значение |
|--------|----------|
| `draft` | Создан |
| `in_progress` | В работе (autopilot — сразу) |
| `partial` | Прерван частично — остаётся в `plans/` |
| `completed` | Готов → переносится в `completed_plans/` |
| `cancelled` | Отменён явно — остаётся в `plans/` |

**Только `completed` переносятся в `completed_plans/`.**

## Изоляция фич
- Ветки: `feature/<area>`. Не трогать файлы чужой фичи.
- Нужен чужой компонент — импортировать, не копировать.

## Контекст и /clear
- `/clear` перед новой задачей. После 3 PR подряд — рекомендовать `/clear`.
- Перегруженный контекст до плана — попросить `/clear`.
