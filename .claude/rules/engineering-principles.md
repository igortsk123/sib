# Инженерная конституция (always-on)

Владелец **не ревьюит код построчно.** Поэтому работа обязана быть **самопроверяемой**: качество держится
на автоматических проверках и явных решениях, а не на ручном просмотре. Обоснование — ADR D3
(`.memory_bank/decisions.md`); чеклист — `.memory_bank/quality-criteria.md`.

## 1. Гипотезы, не аксиомы
- Любой контракт / схема БД / формат письма / правило страховой / выбор библиотеки — это **допущение**.
- Переформулируй как гипотезу → **проверь** (на реальных тестовых письмах / типами / тестом).
- Не совпало — **отклонись** от спеки и запиши **ADR** (`decisions.md`: контекст → решение → последствия).
  Спека (`domain/product-spec.md`) — идея v0.1, не истина (`source-of-truth.md`).

## 2. Маленькие вертикальные срезы
- Один срез = **одна пользовательская способность от UI до БД** (не «весь модуль/Collector сразу»).
- Срез включает: UI-состояние, server action / Inngest-функцию, доступ к БД, **observability** (лог/трейс/метрика).
- Срез **не готов**, пока не проходит проверки (§4).

## 3. Тест рядом с кодом, не «потом»
- Для каждого среза: **unit** на чистую логику (парсинг, нормализация, сопоставление) + **e2e** на
  затронутый пользовательский поток.
- Нет e2e на критический поток → **срез не готов**. Тесты пишутся вместе со срезом, не отдельной задачей.

## 4. Гейт готовности (перед словом «готово»)
Прогнать **весь** гейт: `typecheck` + `lint` + `unit` + `integration` + `e2e` + `build`.
**Любой красный = НЕ готово.** Не объявляй «готово» на красном или непрогнанном гейте.

## 5. Внешний вызов — всегда в обёртке
Никогда не оставляй вызов IMAP / OpenAI / файлового хранилища / любой сети «голым». Обёртка даёт:
- **типизированный результат** (`Result` / discriminated union — без сырых исключений наружу);
- **ретрай через Inngest** (идемпотентные шаги, backoff);
- **заданное user-facing состояние ошибки** (что увидит сотрудник, а не stack trace).

## 6. Спрашивай только по списку
Эскалируй владельцу **только** вопросы из `core/human-decisions.md`. Остальное — **решай сам** и фиксируй
(ADR / `core/*` / `project-state.md`). Не из списка и не тупик → реши как обратимую гипотезу, проверь.

## 7. Завершение среза
Каждый PR/срез заканчивается **зелёным CI** + **обновлённой памятью**, если что-то отклонилось от спеки
(новый/изменённый ADR, правка `core/*`, отметка в `project-state.md`). План → деплой по `agent-workflow.md`.

> Нарушение этих пунктов — не «оптимизация», а дефект процесса (`source-of-truth.md`).
