# Стандарты кода (TS/Next.js)

> Тонкая версия для авто-загрузки. Процесс-конституция — `engineering-principles.md`. Чеклист качества —
> `.memory_bank/quality-criteria.md`.

## Типобезопасность (приоритет — «TS, чтобы ловить»)
- Строгая типизация, **без `any`** и неявных типов. `strict: true`. Внешние данные (письма, JSON, LLM-ответы)
  валидировать на границе (zod/схема) — не доверять форме.
- Named exports (не default). DTO/типы совпадают с контрактами БД (Drizzle-инференс) и внешних API.

## Стиль типизации (ADR D7 — минимум «магии»)
- **Простые, читаемые типы.** Понятные DTO — явные `type`/`interface`, а не выводимые цепочки на полэкрана.
- **Zod на границах:** входящие письма/вложения, ответы OpenAI, формы, env, payload'ы Inngest. TS-тип из
  схемы — `z.infer`; для БД — `drizzle-zod`. Не дублировать схему и тип руками.
- **Без сверхсложных generic** без необходимости; никакой «магии», которую нельзя прочитать за 10 секунд.
- Клиент↔сервер — Next App Router (Server Actions / Route Handlers) с явными DTO. **tRPC не по умолчанию** (D7).

## Слои
- Сетевые/внешние вызовы — в выделенном слое (`lib/`/`services/`), **не в компонентах** и не «голыми»
  (обёртка: типизированный результат + Inngest-ретрай + user-facing ошибка — `engineering-principles.md` §5).
- Тяжёлое (IMAP, OCR, LLM, парсинг вложений) — в **Inngest-функциях**, не в request-пути / server action напрямую.
- Бизнес-логика отделена от разметки и от доступа к БД (тестируемость unit).

## Именование
| Сущность | Конвенция |
|----------|-----------|
| Компоненты/типы | PascalCase |
| Функции/переменные | camelCase |
| Константы | SCREAMING_SNAKE_CASE |
| Файлы-модули | по конвенции Next / feature |

## Тесты рядом
- Чистая логика (парсинг пересылки, нормализация дат/полисов, сопоставление архив↔пароль, дедуп) —
  покрыта **unit**. Критический поток среза — **e2e**. См. `engineering-principles.md` §3.

## Запреты
- Дублирование >10 строк (вынести). Логика в разметке. Оставленные `console.log`. Правка чужой фичи без нужды.
- **Секреты/ПДн в коде, логах, коммитах** — нельзя (`.claude/rules/guardrails.md`).

## Лимиты размера
- Компонент >150 строк → разбить. Модуль/сервис >100 строк → разбить.

## Состояния экрана (UI)
Покрыть `loading` / `error+retry` / `empty` / `success` — не только happy-path (`ui-rules.md`).
