# Guardrails — секреты, ПДн, боевое

**Приоритет: ABSOLUTE.** Задевает защищённое — STOP, переработать план. (Прод ещё нет — фокус на
секретах и персональных/медицинских данных; раздел «боевой флоу» наполнить при запуске.)

## Секреты — НИКОГДА
- Не печатать/коммитить/логировать содержимое `.env*`, `.memory_bank/_secrets/**`, токены Яндекса/OpenAI/Inngest.
- В память (`.memory_bank/`) секреты не кладём — только в `_secrets/ACCESS.md` (600) и `.env.local` (gitignored).
- Реальные креды в коде/доках/тестах — запрещено. Для тестов — фикстуры/моки.

## ПДн и медицинская тайна (ADR D4)
- Письма/вложения = персональные + медицинские данные. Не выводить ФИО/полисы/диагнозы в логи и в чат.
- Логи — со структурным **маскированием PII**. Файлы вложений — в защищённое хранилище вне git (`/storage`, `/uploads`).
- Массовая выгрузка/экспорт — только по правам (`core/roles-and-access.md`).

## Что МОЖНО свободно
- Новые файлы вне секретов/ПДн. Память (`.memory_bank/`), доки, тесты с фикстурами.

## Что С ОСТОРОЖНОСТЬЮ
- Shared-типы / схема БД — аддитивно (миграции expand-only, см. `quality-criteria.md`, `core/data-model.md`).

## Чеклист перед правкой
1. Печатает/коммитит секрет или ПДн? → STOP.
2. Кладёт креды в код/память/тест? → STOP, вынести в env.
3. (При проде) задевает живой флоу/endpoint? → новый, не трогать существующий.

## Когда раздел «боевое» наполняется
При первом проде — добавить prod-URL, защищённые endpoint'ы/файлы, smoke-проверки и откат (`deployment.md`).
